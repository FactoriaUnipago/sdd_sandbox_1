### Post-sync verification (MANDATORY)

After ANY `sdd-sync.sh` execution (sync, refresh, role change, force), ALWAYS verify:

1. ÔśÉ Check exit code ÔÇö 0 = success, non-zero = report error
2. ÔśÉ Run `git status` ÔÇö show what files were added/modified/deleted
3. ÔśÉ Summarize changes to user:
   - New files added (steering, powers, hooks)
   - Files updated (compare versions if possible)
   - Role/config changes applied
   - Stacks synced
4. ÔśÉ If `.sdd-config.json` changed Ôćĺ re-read config values (role, verbosity, stacks)
5. ÔśÉ If powers changed Ôćĺ inform which powers were added/updated
6. ÔśÉ **Compliance check on existing specs** Ôćĺ If templates changed, scan `specs/` for files that now violate the updated template structure (e.g., renamed sections, removed duplicates). Fix them and commit.

ÔÜá´ŞĆ Do NOT just run the command and move on. The user needs to know what changed.

### Sync command differences

| Command | What it does | When to use |
|---|---|---|
| `--refresh` | Re-registers MCPs, re-packages config. Does NOT download new standard files | MCP connection issues, config corruption |
| `--force` | Downloads and overwrites ALL standard files from source | User says "actualiza", "update rules", "sync standard" |
| `--role X` | Changes active role + re-syncs powers for that role | Switching lifecycle phase |

ÔÜá´ŞĆ When user says "actualiza las reglas" or "update standards" Ôćĺ use `--force`, NOT `--refresh`.

### MCP failure rules

On MCP error (`ECONNRESET`, `ECONNREFUSED`, timeout, auth):
1. **REPORT** exact error to user
2. **NEVER** inject env vars, modify config/args, or "temporarily fix"
3. **No exception.** Suggest `bash sdd-sync.sh --refresh` or check credentials.

Prohibited reasoning: "fix connection by adding env var" / "temporary change" / "MCP needs this variable" Ôćĺ ALL BLOCKED.

| Credential | Role | MCP |

### Credential behavior rules

1. **Never block** on optional credentials ÔÇö skip if not needed for current task
2. **Ask only when relevant** ÔÇö one credential at a time, not upfront
3. **After saving** Ôćĺ always `bash sdd-sync.sh --refresh`
4. **Never display saved secrets**
