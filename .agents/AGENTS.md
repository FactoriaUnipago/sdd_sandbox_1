### ⚠️ HARD GATE — One file at a time

**NEVER create the next lifecycle file without explicit user approval.** Each spec file is a STOP point:
1. Create ONE file (e.g., design.md)
2. Show: `✅ Guardado → 📄 [file]` + ask `¿Apruebas?`

**Approval words**: "dale", "apruebo", "ok", "sí", "lgtm", "next", `- [x]`.
**NOT approval**: item selection, branch navigation, "vamos", greetings.

**Item selection ≠ action.** "vamos a ese", "ese", "el primero" = checkout + show context + STOP. Do NOT auto-create any file.

### ⚠️ HARD GATE — Task execution

**Explaining what you WILL do ≠ approval to START doing it.** Before executing ANY task (npm init, creating files, running commands):
1. DESCRIBE the plan: "Haré X, Y, Z"
2. ASK: "¿Procedo?" or "¿Le damos?"
3. **STOP. WAIT for explicit go.**
4. Only after "dale", "sí", "go", "procede" → execute

❌ NEVER run `npm init`, `npx`, `docker`, `cdk`, `git checkout -b`, or any project-modifying command without explicit approval.
❌ "Si me das el go, haré X" is a DESCRIPTION, not approval. The user must RESPOND with approval.

**⚠️ ADO WI creation rules** (see `rules/azure-devops-workflow.md` for details):
- **Duplicate check**: Before creating ANY WI, query ADO with WIQL `CONTAINS` (case-insensitive). If found → do NOT create.
- **Parent link**: Use `wit_work_items_link` (NOT `System.Parent`). Type: `System.LinkTypes.Hierarchy-Reverse`.
- **Brackets lowercase**: ALL text inside brackets `[]` in titles MUST be lowercase: `[producto] [módulo]`, NOT `[Producto]`.
- **HTML tables**: Use `format: "HTML"` for MD/SVC/ED types. Inline styles on EVERY `<td>` and `<th>`: `style="border:1px solid #ddd;padding:6px;"`.

**ADO content signature:** Every text field written to ADO MUST end with: `— 📋 SDD · [power-name] · [YYYY-MM-DD]`

**Spec provenance:** Every spec file MUST start with: `> 📋 Generated by [power] · [date]` + `> ✅ Approved by: [pending | name (date)]`

**MD→HTML:** Description and Acceptance Criteria use HTML. Convert markdown to HTML before writing. Comments support markdown natively.

**docs_language:** Section headers (`## Traceability`, `## Assumptions`, etc.) ALWAYS stay in English (they match `core/templates/` structure). Content inside sections, ADO fields, and output reports use `docs_language` from `.sdd-config.json`. EARS keywords (SHALL/WHEN/WHILE/WHERE/IF) and technical terms stay English. Chat follows user language.

**Rollback:** Tests fail / CR rejected / Deploy fails → In Progress, assign developer. Reqs rejected → Proposed.

**Assignment (2 asks + inherit):** WI created → Creator. Reqs approved → ask developer. All tasks done → ask QA. Everything else inherits. role=all → skip. Save to memory.

**Branch type:** feature/fix → full lifecycle. hotfix → ask dev at WI creation (fast-track). release → creator owns, no asks.
