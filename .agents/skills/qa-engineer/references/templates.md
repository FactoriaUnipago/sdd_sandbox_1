# QA Engineer Templates

## Traceability Header

Include in `test-plan.md`:

```markdown
## Traceability
- **Work Item**: [{prefix}{ID}]({tracker_url})
- **Branch**: `feature/{prefix}{ID}-name`
- **Requirements**: [requirements.md](./requirements.md)
- **Design**: [design.md](./design.md)
- **Tasks**: [tasks.md](./tasks.md)
```

Tracker URL format: Azure DevOps → `https://dev.azure.com/{org}/{project}/_workitems/edit/{ID}`. Others → use URL from `tracker-mapping.md`.

## Test Case Format

Each test case MUST use this format:

```markdown
### TC-001 — [Test title]
- **REQ**: REQ-001
- **Type**: Functional E2E | API | etc.
- **Priority**: 🔴 Mandatory | 🟡 Recommended | 🟢 Optional
- **Steps**: [numbered steps]
- **Expected**: [expected result]
```

## Footer (all spec files)

```markdown
---
> 📍 [{prefix}{ID}]({tracker_url}) · 🌿 `feature/{prefix}{ID}-name` · Generated by SDD Standard
```

## Pending Design Section

When design is NOT available, mark technical sections:

```
## ⏳ Pending — requires design

The following sections will be completed when the design is available:
- [ ] API Testing — endpoints and contracts
- [ ] Performance Testing — targets and thresholds
- [ ] Integration Testing — components and dependencies
- [ ] Contract Testing — schemas
```
