# API Documentation — Universal

Every backend MUST serve OpenAPI spec + interactive documentation. Zero or minimal dependencies.

## Requirements

1. **OpenAPI spec** → `docs/api/openapi.yaml` (repo root). Generated by `doc-generator` from `design.md §3.3`.
2. **Swagger UI** → served at `/api/docs` — interactive, "Try it out" for testing endpoints.
3. **ReDoc** → served at `/api/redoc` — clean three-column reading format (optional but recommended).
4. **Raw spec endpoint** → served at `/api/docs/spec` (YAML or JSON).
5. **Public access** → API docs routes MUST be outside auth middleware.
6. **Non-production toggle** → optionally disable in production via env var `API_DOCS_ENABLED=true|false`.

## Implementation by Stack

### Node / Express

Zero deps — Swagger UI + ReDoc via CDN. See `stacks/node/express-patterns.md` §API Documentation.

```ts
// GET /api/docs     → Swagger UI (CDN) — interactive
// GET /api/redoc    → ReDoc (CDN) — reading format
// GET /api/docs/spec → openapi.yaml
```

### Node / NestJS

Built-in `@nestjs/swagger` module:

```ts
// main.ts
import { SwaggerModule, DocumentBuilder } from '@nestjs/swagger';

const config = new DocumentBuilder()
  .setTitle('API')
  .setVersion('1.0')
  .addBearerAuth()
  .build();
const doc = SwaggerModule.createDocument(app, config);
SwaggerModule.setup('api/docs', app, doc);
```

### Python / FastAPI

Built-in — automatic. No config needed:

```python
# FastAPI serves docs automatically:
# GET /docs     → Swagger UI
# GET /redoc    → ReDoc
# GET /openapi.json → spec

# To customize:
app = FastAPI(
    title="API",
    docs_url="/api/docs",
    redoc_url="/api/redoc",
    openapi_url="/api/docs/spec",
)
```

### Python / Django

Use `drf-spectacular`:

```python
# settings.py
INSTALLED_APPS += ['drf_spectacular']
REST_FRAMEWORK = {'DEFAULT_SCHEMA_CLASS': 'drf_spectacular.openapi.AutoSchema'}

# urls.py
from drf_spectacular.views import SpectacularAPIView, SpectacularRedocView
urlpatterns += [
    path('api/docs/spec', SpectacularAPIView.as_view(), name='schema'),
    path('api/docs', SpectacularRedocView.as_view(url_name='schema'), name='redoc'),
]
```

### .NET / ASP.NET Core

Built-in Swagger via `Swashbuckle`:

```csharp
// Program.cs
builder.Services.AddEndpointsApiExplorer();
builder.Services.AddSwaggerGen();

// ...
app.UseSwagger();         // GET /swagger/v1/swagger.json
app.UseSwaggerUI(c => {   // GET /api/docs
    c.SwaggerEndpoint("/swagger/v1/swagger.json", "API v1");
    c.RoutePrefix = "api/docs";
});
```

### Java / Spring Boot

Built-in via `springdoc-openapi`:

```xml
<!-- pom.xml -->
<dependency>
    <groupId>org.springdoc</groupId>
    <artifactId>springdoc-openapi-starter-webmvc-ui</artifactId>
    <version>2.6.0</version>
</dependency>
```

```yaml
# application.yml
springdoc:
  api-docs:
    path: /api/docs/spec
  swagger-ui:
    path: /api/docs
```

## Rules

1. **Every backend serves API docs** — no exceptions. If the framework has built-in support, use it.
2. **Prefer zero-dep solutions** — CDN for ReDoc/Swagger UI when possible.
3. **Consistent routes** — `/api/docs` for UI, `/api/docs/spec` for raw spec.
4. **Public access** — outside auth. Register alongside health routes.
5. **Spec source** — `docs/api/openapi.yaml` is the single source of truth, generated from design.md §3.3.
6. **Include in Wave 1** — API docs setup is part of server/framework initialization, not deferred.
