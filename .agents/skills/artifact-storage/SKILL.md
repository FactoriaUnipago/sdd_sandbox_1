# Artifact Storage

Repo = versionable (specs, tests, docs, migrations, pipelines). Azure = live state (boards, PRs, test runs). Gitignored = ephemeral (reports, credentials, cache).

## Project structure

> Full tree: `references/project-tree.md`

Key paths:

| Path | Generator | Contents |
|---|---|---|
| `specs/{prefix}[ID]-[name]/` | analyst/designer/QA | requirements, design, tasks, test-plan, migration-plan |
| `tests/postman/collections/` | api-tester | `[project-name].json` (1 collection, folders per feature) |
| `tests/postman/environments/` | api-tester | dev.json, cert.json, prod.json |
| `tests/e2e/` | qa-engineer | `{prefix}[ID]-[name].spec.ts` (Playwright) |
| `tests/data/` | test-data-generator | `{prefix}[ID]-seed.sql` |
| `tests/reports/` | — | ⚠️ GITIGNORED |
| `docs/api/` | doc-generator | openapi.yaml |
| `docs/architecture/` | scanner/designer | ⚠️ **GLOBAL** — system-design, data-model, api-contract, security-model, integrations. Accumulates from ALL features. NEVER feature-specific. |
| `docs/diagrams/` | solution-designer | infrastructure.py/.png, `{prefix}[ID]-[name].svg` |
| `docs/releases/` | release-notes-writer | vX.Y.Z.md |
| `db/migrations/` | db-migrator | NNN_description.sql |
| `.azure-pipelines/` | pipeline-builder | CI/CD YAML |
| `.sdd-config.json` | scanner/designer | Project config |
| `.sdd-credentials.json` | — | ⚠️ GITIGNORED |

## Azure DevOps (not in repo)

| Artifact | Power | Destination |
|---|---|---|
| Work item updates | All | Work Items |
| Test results | qa-engineer | Test Plans → Test Runs |
| Code review | code-reviewer | PR Comments |
| Sprint plan | sprint-planner | Sprint Board |
| Regression metrics | regression-analyst | Dashboards |

## Storage rules

| Type | Repo | Gitignored | Azure DevOps |
|---|:---:|:---:|:---:|
| Specs, Postman, E2E, test data, API docs, diagrams, DB migrations, pipeline YAML | ✅ | — | — |
| Test reports/results | — | ✅ | ✅ Test Runs |
| Release notes | ✅ | — | ✅ Comment |
| Code reviews, sprint plans, regression metrics | — | — | ✅ |
| Credentials, cache (.sdd-cache) | — | ✅ | — |

## File editability

| File | Type | Editable | Notes |
|---|---|:---:|---|
| `product.md` | LOCAL | ✅ | Project identity — never overwritten by sync |
| `design-system-theme.md` | LOCAL | ✅ | Selected theme |
| `.sdd-config.json` | LOCAL | ✅ | Enriched by scanner/designer |
| `.sdd-credentials.json` | LOCAL | ✅ | Gitignored |
| `specs/**/*.md` | PROJECT | ✅ | By assigned role |
| `skills/*.md` (except product.md) | STANDARD | 🚫 | Managed by `sdd-sync.sh` |
| `powers/**`, `hooks/**`, `.githooks/*` | STANDARD | 🚫 | Managed by `sdd-sync.sh` |
| `settings/mcp.json`, `.sdd-standards-version` | STANDARD | 🚫 | Generated by `sdd-sync.sh` |

**product.md is NOT a standard file.** Any role can modify it. Do NOT block commits.

**STANDARD files**: ✅ commit them (essential for team) · 🚫 do NOT edit manually (overwritten on sync). To edit: `bash sdd-sync.sh --force`

**Uncommitted STANDARD files** → `git add -A && git commit --no-verify -m "chore: update SDD standard files"`

## NEVER

- NEVER store test reports in repo (→ Azure DevOps Test Runs)
- NEVER store credentials outside `.sdd-credentials.json`
- NEVER create artifacts outside defined structure
- ALWAYS use {prefix}ID in file name for traceability
- NEVER block commits to LOCAL files (product.md, .sdd-config.json, specs/)
