# Deployment Guide — AWS RDS PostgreSQL

> 📋 Generated by `solution-designer` · 2026-07-08

Esta guía detalla los pasos para aprovisionar y gestionar la base de datos PostgreSQL 16 remota en AWS RDS utilizando **AWS CDK** en TypeScript.

## Prerequisites

Antes de iniciar el despliegue, asegúrate de contar con los siguientes elementos instalados y configurados en tu máquina:

1. **AWS CLI** configurado con credenciales válidas y permisos para crear recursos (VPC, Subnets, RDS, Secrets Manager, etc.):
   ```bash
   aws configure
   ```
2. **Node.js** (v18 o superior) y **npm** instalados.
3. El Toolkit de AWS CDK instalado de forma global o ejecutable localmente vía `npx`:
   ```bash
   npm install -g aws-cdk
   ```

## Deployment Steps

Sigue los siguientes comandos desde la carpeta de infraestructura del proyecto (`app/infra/`) para realizar el despliegue:

### 1. Bootstrap de la cuenta de AWS
El bootstrapping es un proceso obligatorio que prepara la cuenta de AWS y la región seleccionada para recibir el despliegue de CDK (crea un bucket de S3 para assets y roles IAM asociados):
```bash
npx cdk bootstrap
```

### 2. Sintetizar la plantilla CloudFormation
Genera el archivo CloudFormation equivalente en JSON/YAML a partir de nuestro código de CDK. Esto sirve para verificar la sintaxis y asegurar que la plantilla compile limpiamente:
```bash
npx cdk synth
```

### 3. Desplegar los recursos en AWS
Este comando inicia la creación de la base de datos RDS, la VPC, y los grupos de seguridad configurados en AWS:
```bash
npx cdk deploy
```
*Durante el despliegue, CDK te pedirá confirmación de los cambios de seguridad (Security Group e IAM Roles). Escribe `y` para proceder.*

### 4. Destruir los recursos (Cleanup)
Si necesitas dar de baja la base de datos y limpiar todos los recursos de la cuenta para no incurrir en costos continuos:
```bash
npx cdk destroy
```

---

## Verification

Una vez que `cdk deploy` finalice exitosamente, se mostrarán las salidas del stack en la consola (Outputs). Deberás registrar los siguientes datos:

1. **DatabaseEndpoint**: La dirección URL de conexión del host de la base de datos.
2. **DatabasePort**: El puerto (por defecto `5432`).
3. **CredentialsSecretArn**: El identificador ARN en Secrets Manager donde se guardaron la contraseña y usuario generados aleatoriamente (`dbadmin`).

Para recuperar las credenciales generadas de forma segura:
```bash
aws secretsmanager get-secret-value --secret-id task-manager-db-credentials --query SecretString --output text
```

## Configuration

Para conectar el servidor de producción a la base de datos de AWS RDS, copia la plantilla `.env.production.example` a un archivo `.env` en producción y sustituye la variable `DATABASE_URL` con el formato obtenido en la sección anterior:

```ini
DATABASE_URL="postgresql://dbadmin:PASSWORD_SECRETO@DATABASE_ENDPOINT:5432/taskdb?sslmode=require"
```
